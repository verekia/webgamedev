## 🔥 Headlines

😺 [**Crashcat**](https://crashcat.dev/) is [Isaac Mason](https://x.com/isaac_mason_)'s new physics engine. With rigid body simulation, raycasts, shapecasts, constraints, and CCD, it's a tree-shakable and library agnostic pure JavaScript engine. See how it compares with other physics engine on Isaac's [benchmark demo](https://isaac-mason.github.io/js-physics-benchmarks/) page. And check out [**Navcat**](https://navcat.dev/) for your pathfinding needs if you haven't already!

<WebGPULogo /> [**Electrobun**](https://blackboard.sh/electrobun/docs/) – the Bun alternative to
Electron – now supports [**wgpu**](https://github.com/gfx-rs/wgpu), the Rust WebGPU API
([announcement](https://blackboard.sh/blog/wgpu-in-electrobun/)). This bridges the gap between JS
and native WebGPU runtime via adapters. See the
[Three.js](https://github.com/blackboardsh/electrobun/tree/main/templates/wgpu-threejs) and
[Babylon.js](https://github.com/blackboardsh/electrobun/tree/main/templates/wgpu-babylon) examples.
If you give it a shot, please share your experience on [our
Discord](https://webgamedev.com/discord)!

<ThreeLogo /> [**Three VFX**](https://three-vfx.vercel.app/)
([repo](https://github.com/mustache-dev/Three-VFX)) is a feature-packed GPU-accelerated particle
system with support for React Three Fiber, TresJS, Threlte, and vanilla Three.js by
[Alex](https://x.com/mustache_dev) and [me](https://x.com/verekia). It supports emitter shapes,
physics and collisions, bezier curves, soft particles, attractors and vortices, trails, flipbook and
textures, and a lot more.
