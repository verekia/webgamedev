## üî• Headlines

<WebGLLogo /> <WebGPULogo /> The Khronos WebGL + WebGPU Meetup took place last week. Here are the
[slides](https://khr.io/web202301), [video](https://www.youtube.com/watch?v=Jl06sOvMnvU), and
[highlights](https://omar-shehata.medium.com/webgl-webgpu-jan-2023-meetup-highlights-d29f9e0d0bdc).

My main personal takeaway from the meetup is that Three.js' [mrdoob](https://twitter.com/mrdoob) thinks the way forward for writing shaders is node-based solutions that provide a compatibility abstraction layer over WebGL Shader Language (GLSL) and WebGPU Shader Language (WGSL). In Three.js, that would be the [**NodeMaterial**](https://www.donmccurdy.com/2019/03/17/three-nodematerial-introduction/), which has a [visual editor](https://threejs.org/examples/webgl_nodes_playground.html). Three.js is working on aligning with the [**MaterialX**](https://materialx.org/) open standard for node-based materials and already has a [MaterialXLoader](https://threejs.org/examples/?q=materialx#webgl_nodes_loader_materialx). [Sunag](https://github.com/sunag) does the heavy lifting on that front. You can also check out [Hendrik Mans](https://www.webgamedev.com/interviews/hendrik-mans)' [**Shader Composer**](https://github.com/hmans/composer-suite/tree/main/packages/shader-composer), which has React Three Fiber bindings but currently only supports GLSL.

<TresLogo /> [Alvaro Sabu released](https://twitter.com/alvarosabu/status/1620836387925925890)
[**TresJS**](https://tresjs.org/), a library to create Three.js scenes declaratively with
[**Vue**](https://vuejs.org/). Read the [Motivation](https://tresjs.org/guide/#motivation) section
to see how it relates to [**TroisJS**](https://troisjs.github.io/) and
[**Lunchbox**](https://github.com/breakfast-studio/lunchboxjs), which are similar Vue wrappers for
Three.js.

<WebXRLogo /> Meta released the [**Immersive Web
Emulator**](https://twitter.com/Oculus_Dev/status/1622672216348573697) to easily test WebXR
experiences on desktop ([Chrome
Store](https://chrome.google.com/webstore/detail/immersive-web-emulator/cgffilbpcibhmcfbgggfhfolhkfbhmik)).

## üõ† Tools & Updates

- <ThreeLogo /> [**Three.js**](https://threejs.org/) only supports instanced skinned meshes [for
  WebGPU](https://threejs.org/examples/#webgpu_skinning_instancing), not WebGL, so [Cody
  Bennett](https://twitter.com/Cody_J_Bennett) wrote a user-land port of the [unmerged
  PR](https://github.com/mrdoob/three.js/pull/22667) for **InstancedSkinnedMesh** in a
  [CodeSandbox](https://codesandbox.io/s/three-js-instanced-skinned-mesh-2yfgiu).
- <ThreeLogo /> [**Editor and code
  generator**](https://twitter.com/a_warnes/status/1620664524591472641) for
  **MeshTransmissionMaterial** by [Alex Warnes](https://twitter.com/a_warnes). Works for R3F and
  Threlte.
- <ThrelteLogo /> [**@threlte/gltf**](https://github.com/threlte/threlte/tree/main/packages/gltf) is
  an experimental fork of [gltfjsx](https://github.com/pmndrs/gltfjsx) for Threlte by [Grischa
  Erbe](https://twitter.com/threlte) to transform gLTF and GLB files into declarative Threlte
  components
  ([demo](https://stackblitz.com/edit/test-threlte-gltf?file=src%2Flib%2FGameboy.svelte)).
- <ThreeLogo /> [**Phy**](https://github.com/lo-th/phy), [3TH](https://twitter.com/3dflashlo)'s game
  engine to add physics to Three.js, now supports [NVIDIA
  PhysX](https://developer.nvidia.com/physx-sdk) 5.1 via
  [physx-js-webidl](https://github.com/fabmax/physx-js-webidl) JavaScript / Wasm bindings. See the
  [other new features](https://twitter.com/3dflashlo/status/1621576273100242944) and [live
  demo](https://lo-th.github.io/phy/). 3TH also created [Oimo.js](https://github.com/lo-th/Oimo.js)
  and [Full IK](https://github.com/lo-th/fullik).
- <PlayCanvasLogo /> [**PlayCanvas v1.61.0**](https://github.com/playcanvas/engine/releases) ‚Äì
  PlayCanvas now supports Microsoft's [**Volumetric
  Videos**](https://twitter.com/playcanvas/status/1620795982337421312). All PlayCanvas builds now
  include **WebGPU support** (it was previously only included in debug builds), adding only
  [9kb](https://github.com/playcanvas/engine/pull/5030) to the engine!
- <BabylonLogo /> [**Babylon.js v5.45.0**](https://github.com/BabylonJS/Babylon.js/releases)
