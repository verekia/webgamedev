## üî• Headlines

<DiscordIcon className="size-5 inline-flex align-middle relative -top-[2px]" /> **Discord** shared their
plan for the [future](https://discord.com/blog/discords-next-chapter) and launched their [**Embedded
App SDK**](https://discord.com/build/embedded-app-sdk) for Activities ([NPM package](https://www.npmjs.com/package/@discord/embedded-app-sdk)).
Multiplayer web games can now be played directly within Discord via iframes, with users connected via
voice chat. The SDK lets you authenticate players and manage how your game interacts with Discord. Check
out the [official starter](https://github.com/discord/embedded-app-sdk/tree/a124ae8eaf51c3fc118059e76af89b051c1c6277/examples/discord-activity-starter),
[Hugo Duprez](https://x.com/HugoDuprez)'s [starter](https://github.com/Hugo-Dz/discord-activity-starter),
and [Colyseus](https://colyseus.io/)' [starter](https://github.com/colyseus/discord-embedded-app-sdk)
([article](https://colyseus.io/blog/discord-embedded-sdk/)).

üçè **Apple**'s [WWDC](https://developer.apple.com/wwdc24/) developer event is ongoing, where [**Safari 18 beta**](https://webkit.org/blog/15443/news-from-wwdc24-webkit-in-safari-18-beta/) got announced. Even though WebGPU has been available in the [Technology Preview](https://www.webkit.org/blog/14879/webgpu-now-available-for-testing-in-safari-technology-preview/) since last December, it is unfortunately not part of this release. So you will have to support WebGL for a while if you support Safari. On the Web Apps ("PWAs") front, if a user has your web app installed and they click a link pointing to your domain from _outside of their browser_ (like a link shared on Discord or Slack apps), it will open it the web app. There are still no [Install Prompts](https://web.dev/learn/pwa/installation-prompt) on Safari, so you will have to keep explaining to your users how to manually install your web app. Ô∏èFinally, <WebXRLogo /> **WebXR** is no longer behind a feature flag, and `immersive-vr` sessions and hand tracking are [available by default](https://x.com/jamesckane/status/1800564251037495352).

<WebGPULogo /> The **Meta Quest Browser** now [**supports
WebGPU**](https://developer.oculus.com/documentation/web/browser-release-notes/) since version 32.0!

7.0**](https://babylonjs.medium.com/introducing-babylon-js-7-0-a141cd7ede0d) is out! Check out the

Physics](https://doc.babylonjs.com/features/featuresDeepDive/physics/ragdolls) and better WebXR
support.
